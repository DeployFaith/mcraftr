# Mcraftr Roadmap

This is a lightweight roadmap for the near-term direction of the project.

## Now

- stabilize the public demo experience
- improve README, install, and Full Stack explanation
- keep screenshot/demo assets fresh and credible
- reduce CI/image-build turnaround time
- tighten core Playwright coverage for admin-critical flows

## Next

- document the exact value of Bridge + Beacon over plain RCON
- improve contribution onboarding and issue hygiene
- publish an initial tagged release
- record a short product walkthrough demo
- expand demo-safe seeded content for screenshots and try-now flows

## Later

- deeper world-aware workflows for Full Stack installs
- more robust scheduling and audit tooling
- better live player context and moderation shortcuts
- stronger multi-server operational UX
- more guided first-run onboarding

## Good First Contributions

- README clarity improvements
- install docs fixes
- Playwright regression coverage for visible bugs
- screenshot/demo automation improvements
- UI polish that reduces clutter in high-traffic admin views

## Help Wanted Areas

- Full Stack docs and onboarding
- demo environment reliability
- CI/build optimization
- high-signal feature requests backed by real admin workflows
