# Mcraftr Roadmap

Mcraftr is now publicly tagged and released at `v0.2.3`.

This roadmap tracks the most relevant near-term priorities after the first real release.

## Now

- follow up quickly on post-release bug reports and self-host friction
- turn the disposable Playwright harness work into a cleaner first-class local workflow in the main repo
- tighten core Playwright coverage for admin-critical flows and local production-mode smoke checks
- reduce CI and image-build turnaround time
- keep screenshots and product visuals fresh and credible for the released product

## Next

- document the exact value of Relay + Beacon over plain RCON with clearer operator examples
- improve contribution onboarding and issue hygiene for outside contributors
- expand seeded content for screenshots, walkthroughs, and release demos
- smooth rough edges in Full Stack setup and capability messaging
- chip away at the current lint and React-effect warning backlog without destabilizing release behavior

## Later

- deeper world-aware workflows for Full Stack installs
- more robust scheduling and audit tooling
- better live player context and moderation shortcuts
- stronger multi-server operational UX
- more guided first-run onboarding

## Good First Contributions

- Playwright regression coverage for visible bugs
- screenshot automation improvements
- contribution and setup docs polish
- UI polish that reduces clutter in high-traffic admin views
- targeted cleanup of unused vars and obvious warning-level lint issues

## Help Wanted Areas

- Full Stack docs and onboarding
- CI and build optimization
- high-signal feature requests backed by real admin workflows
- reproducible self-host bug reports with environment details
